Introduction
The legacy i18n message ID is not stable when there are insignificant changes, such as to whitespace.
  • The new canonical will reduce the unnecessary translation invalidation and associated retranslation cost in applications where translations do not match due to whitespace changes for example.
  • After v11, new projects are automatically configured to use the new message ids.
  • For the existed applications, the ng cli can help to migrate the legacy localization IDs to ID from the latest algorithm.
    There a few steps to migarte the legacy localization IDs.
    1. Install the localize package.
    2. Generate the migration file.
    3. Update the IDs according to the migration file
    4. Append the following line to the ~/.zshrc file the terminal show "zsh: no matches found: --files=*.xlf"
