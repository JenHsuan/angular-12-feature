New Features of TypeScript 4.2
There are a few new features in TypeScript 4.2. I pick up some confused topics and introduced them in this section.
  • Smarter Type Alias Preservation: (reference)
    Reduce the size of error messages of hover hints.
  • Leading/Middle Rest Elements in Tuple Types: (reference)
    Now rest elements can occur anywhere within a tuple. But it's restricted to two scenarios.
    1. Unallowed to put the rest elements after another rest elements
    2. Unallowed to put the rest elements in front of the optional parameter
  • --noPropertyAccessFromIndexSignature: (reference)
    We are only allowed to get properties declared by index signature with bracketed element access syntax like person["name"]

  • Destructured Variables Can Be Explicitly Marked as Unused: (reference)
    • Previously, if _first was never used later on, TypeScript would issue an error under noUnusedLocals.
    • Now, TypeScript will recognize that _first was intentionally named with an underscore because there was no intent to use it.
  • abstract Construct Signatures: (reference)
    • We can not assign an abstract class to anything that expects a construct signature. It is overly-restrictive in case we want to write a subclass of Ctor.
    • In v12, we are allowed to specify an abstract modifier on constructor signatures.
