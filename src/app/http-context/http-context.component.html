Introduction
Currently there is no way to configure interceptors in HttpClient on per request basis. Suppose that we have an interceptor to alert the error message if the proxy is not ready.
  • All response will be applied to this interceptor.
  • How do we bypass this interceptor for some requests?
Pass Metadata to Interceptors
In v12, we are allowed to customize HttpInterceptor by passing data through HttpContext. (pull request). There are a few steps to pass metadata to the interceptor.
  1. Create a HttpContextToken
  2. Create a new interceptor
  3. Update the HTTP service.
    • Pass the HttpContextToken to the HttpClient.
    • Set the value to the HttpContextToken.
