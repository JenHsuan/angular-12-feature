Description
Angular v12 enable the strict mode on TypeScript and Angular by default. There are some effects.
  • Enable the strict mode on TypeScript recommended by the TypeScript team.
  • Turn on the strict Angular compiler flag.
TypeScript Strict Mode
  1. forceConsistentCasingInFileNames: when importing modules from the file, the casting of the file name is case-sensitive.
    example:
  2. noImplicitReturns: throw the error message if not all code return value in a function.
    example:
  3. noFallthroughCasesInSwitch: ensures that any non-empty case inside a switch statement includes either break, return, or throw.
    example:
Angular Compiler Flags
  1. strictTemplates: Angular will check the type on the template
    example:
  2. strictInjectionParameters: Angular reports an error for a supplied parameter whose injection type cannot be determined.
  3. strictInputAccessModifiers: Whether access modifiers such as private/protected/readonly are honored when assigning a binding expression to an @Input(). If disabled, the access modifiers of the @Input are ignored; only the type is checked.
    example:
